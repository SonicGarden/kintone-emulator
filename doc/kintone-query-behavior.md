# kintone クエリ挙動メモ（実機観察）

実 kintone のクエリ（`/k/v1/records.json?query=...`）の仕様書に載っていない、または重要な実挙動を記録する。エミュレーターで再現する際の根拠。

観察日時: 2026-04-24

## 文字列リテラル

### エスケープシーケンス

kintone 仕様書には `\"` と `\\` のみ明記されているが、実機は以下のエスケープも受け付ける:

| 記法 | 展開 |
|---|---|
| `\"` | `"` |
| `\\` | `\` |
| `\t` | タブ文字（U+0009） |
| `\n` | （観察上は LF 相当に展開される模様） |
| `\r` | （未検証、仕様上は LF と同様に想定） |

### 生制御文字の禁止

**文字列リテラル内に生のタブ / 改行 / キャリッジリターンを含めるとエラー** になる:

```
POST /k/v1/records.json?query=文字列__1行_ like "<生タブ>"
→ 400 CB_VA01 / "クエリ記法が間違っています。"
```

クライアント側で `\t` などのエスケープに変換してから送る必要がある。

### トークン間の空白

文字列リテラル**以外**の位置（フィールドコードと演算子の間など）では、以下のいずれも空白として許容される:
- 半角スペース、タブ（`\t`）
- 改行（LF, CR, CRLF）
- 全角スペース `　`（U+3000）

つまり空白の扱いは「文字列リテラル内はエラー / 外側は緩い」という非対称構造。

## 文字列比較の挙動

### `like` / `not like` は「単語検索」

仕様書にもあるが、**SQLite の LIKE `%キーワード%` とはセマンティクスが異なる**。実機は形態素解析ベースの全文インデックス検索。

```
データ: "gamma keyword"
クエリ: 文字列__1行_ like "key"     → 0 件（部分文字列では一致しない）
クエリ: 文字列__1行_ like "keyword" → 1 件（単語単位で一致）
```

ひらがな 1 文字（`"a"` など）も、検索インデックスの最小文字数を下回っていると 0 件。

→ **エミュレーターでの完全再現は困難**。現状は SQLite の LIKE `%...%` で代用しているため挙動が異なることを README・ドキュメントに明記する。

### 空値判定

- `is empty` / `is not empty` は MULTI_LINE_TEXT / RICH_TEXT / FILE でのみ利用可
- **空白文字・改行・タブのみの文字列も「空」扱い**（`"   "` や `"\n"` も該当）
- SINGLE_LINE_TEXT 等には `is empty` は使えない

### `in ("")` の挙動

**値を保持していない行 + 空文字を保持する行の両方にヒット**する。つまり kintone 側では「未選択 = null / 空文字 / 空配列」を同一視。

## 演算子のフィールドタイプ制限

仕様書の表通りだが、不一致時のエラー形式:

```
POST /k/v1/records.json?query=ドロップダウン = "ドロップダウン1"
→ 400 GAIA_IQ03 / "ドロップダウンフィールドのフィールドタイプには演算子=を使用できません。"
```

MULTI_LINE_TEXT / RICH_TEXT で `=` を使うと同じく `GAIA_IQ03`。

## エラーレスポンス集

| ケース | code | HTTP | message |
|---|---|---|---|
| 構文エラー | `CB_VA01` | 400 | `入力内容が正しくありません。` + `errors.query.messages = ["クエリ記法が間違っています。"]` |
| 文字列内の生制御文字 | `CB_VA01` | 400 | 同上 |
| `limit` が 500 超 | `GAIA_QU01` | 400 | `limit には 500 以下の値を指定してください。` |
| `limit` が負数 | `CB_VA01` | 400 | `入力内容が正しくありません。` |
| `offset` が 10000 超 | `GAIA_QU02` | 400 | `offset には 10,000 以下の値を指定してください。` |
| 存在しないフィールド | `GAIA_IQ11` | 400 | `指定されたフィールド（<code>）が見つかりません。` |
| 演算子タイプ不一致 | `GAIA_IQ03` | 400 | `<code>フィールドのフィールドタイプには演算子<op>を使用できません。` |
| 関数引数の値が無効 | `GAIA_IQ23` | 400 | `指定した日にち（<val>）は無効です。1から31までの整数で指定します。`（THIS_MONTH 等）|
| 関数引数の unit が無効 | `GAIA_IQ25` | 400 | `FROM_TODAY()の第2引数には、「DAYS」「WEEKS」「MONTHS」「YEARS」のいずれかを指定します。`|
| 関数引数の個数エラー | `GAIA_IL08` | 400 | `クエリの指定が不正です。` |

## オプション

### `limit` / `offset`

- `limit`: `records.json` GET では最大 **500**。省略時 100
- `offset`: 最大 **10000**。省略時 0
- 負値は `CB_VA01`

### デフォルトの並び順

`order by` を省略した場合、**レコード番号（$id）の降順**。

### `totalCount`

GET 時に `?totalCount=true` を付けないと、レスポンスの `totalCount` は `null` として返る。付けると件数が入る。今回調べた範囲ではクエリ内の `order by` / `limit` 等とは独立した URL クエリパラメータ。

## 関数

仕様書にあるものは全て実装可。ただし以下に注意:

### `FROM_TODAY(n, UNIT)`

- 第 1 引数は必須で整数
- 第 2 引数は `DAYS` / `WEEKS` / `MONTHS` / `YEARS` のいずれか（シンボル、クォートなし）
- 引数の個数が足りないと `GAIA_IL08`、UNIT が不正だと `GAIA_IQ25`

### `THIS_WEEK(曜日)` / `LAST_WEEK` / `NEXT_WEEK`

- 曜日は `SUNDAY` ... `SATURDAY`（シンボル、クォートなし）
- 引数なしだと週全体（日〜土）

### `THIS_MONTH(...)` / `LAST_MONTH` / `NEXT_MONTH`

- 引数なし: 月全体
- `LAST` シンボル: 月末日
- 1〜31 の整数: 月内のその日
- 範囲外や無効値は `GAIA_IQ23`
- 月末の丸め: 「その月にその日が存在しなければ翌月 1 日」（例: `THIS_MONTH(31)` で 4 月 → 5 月 1 日にずれる仕様。要検証）

### `NOW()` / `TODAY()` / `YESTERDAY()` / `TOMORROW()` / `THIS_YEAR` / `LAST_YEAR` / `NEXT_YEAR`

引数なし。素直。

### `LOGINUSER()` / `PRIMARY_ORGANIZATION()`

API 呼び出し元のユーザー情報を展開する。**エミュレーターでは認証ユーザー情報をパーサーコンテキストに渡す配線が必要**（現状未実装）。

## 識別子

### フィールドコード

kintone のフィールドコード命名規則:
- 使える文字: 半角英数、半角アンダースコア、ひらがな、カタカナ、漢字、全角英数、全角記号の一部（「・」「＿」等）
- 識別子はクォート不要。**バッククォートも使えない**（実機はバッククォートを識別子クォートとして解釈しない）
- kintone 既定のシステムフィールドコード: `レコード番号` / `作成日時` / `更新日時` / `作成者` / `更新者` / `カテゴリー` / `ステータス` / `作業者`
- `$id` は「レコード番号」の別名として予約（実機でレコード番号を指すクエリで使用）

### 関連レコード / テーブル内フィールド

`関連レコード.フィールド名` 形式でネスト参照可（仕様書）。未検証。
