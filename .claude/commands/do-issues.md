`doc/issues/*.md` にあるファイルをファイル名の昇順に一つずつ処理してください。

## 処理手順

各 issue ファイルに対して以下を順番に実行してください:

### 1. issue の読み込み

`doc/issues/` ディレクトリのファイルをファイル名昇順で列挙し、最初の未処理ファイルを読み込む。

### 2. issue の実装

ファイルに書かれたタスクを実装する。

### 3. タスク完了のワークフロー

`/finish-task` を実行する。

### 4. issue ファイルの移動

完了した issue ファイルを `doc/issues/done/` に移動する。

ファイル名の形式: `YYYY-MM-DD-HH-MM-SS-<issueの内容を表す名前>.md`

- 日時は現在時刻（JST）を使用する
- `<issueの内容を表す名前>` は元のファイル名や内容から適切な英語のケバブケース名を付ける
- 移動には `mv` コマンドを使う

### 5. git commit

実装内容と issue ファイルの移動をまとめて1つのコミットにする。

### 6. 次の issue へ

残りの issue ファイルがあれば、手順 1 に戻って次のファイルを処理する。すべて完了したら終了する。

## 注意事項

- `doc/issues/done/` ディレクトリがなければ事前に作成する
- issue が 0 件の場合は「処理する issue がありません」と報告して終了する
- 1つの issue を完全に完了（ワークフロー含む）してから次に進む
